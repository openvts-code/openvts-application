ISSUES ADDRESSED
================================================================================

Issue #1: Profile Tab City Shows "-"
  Problem:  Empty dash instead of city name
  Cause:    AdminUserAddress.cityName is empty when API only returns cityId
  Fix:      Enhanced parsing to include cityId as fallback in cityName
  Status:   ✅ FIXED

Issue #2: Edit Profile City Dropdown Blank
  Problem:  City field shows no selection even when selected
  Cause:    No fallback from cityId, no injection of current value
  Fix:      Initialize with fallback to cityId, inject current as option
  Status:   ✅ FIXED

Issue #3: Location Codes Instead of Full Names
  Problem:  Show "IN" instead of "India", "MH" instead of "Maharashtra"
  Cause:    _ProfileSnapshot only stores codes, not display names
  Fix:      Added countryName and stateName fields to store display names
  Status:   ✅ FIXED

Issue #4: City ID/Name Mismatch
  Problem:  Edit sheet uses different field sources than dropdown values
  Cause:    Inconsistent data source reconciliation
  Fix:      Unified city initialization with priority logic
  Status:   ✅ FIXED

BEHAVIOR VERIFICATION
================================================================================

Scenario 1: Backend returns cityName
  Input:    address.cityName = "Mumbai", address.cityId = "12345"
  Result:   Display "Mumbai" ✅
  Reason:   _firstNonEmpty prioritizes cityName

Scenario 2: Backend returns only cityId
  Input:    address.cityName = "", address.cityId = "12345"
  Result:   Display "12345" (as fallback) ✅
  Reason:   _firstNonEmpty falls back to cityId

Scenario 3: Edit Profile initialization with cityName
  Input:    profile.city = "Mumbai"
  Result:   _city = "Mumbai", dropdown shows selection ✅
  Reason:   _blankToNull returns "Mumbai", not null

Scenario 4: Edit Profile initialization with cityId
  Input:    profile.city = "", profile.cityId = "12345"
  Result:   _city = "12345", dropdown shows "(current)" option ✅
  Reason:   _city = _blankToNull("") ?? _blankToNull("12345") = "12345"

Scenario 5: Dropdown loads after sheet opens
  Input:    _city = "Mumbai", _cities = [] (loading)
  Result:   Dropdown shows "Mumbai (current)" option ✅
  Reason:   Fallback injection adds current value if not in options

Scenario 6: No city data
  Input:    address.cityName = "", address.cityId = ""
  Result:   Display "-" (empty, correct) ✅
  Reason:   _firstNonEmpty returns "", displayed as dash
